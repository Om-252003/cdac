What is Linux?

Linux is a Unix-like, open-source, and community-developed operating system (OS) for computers, servers, mainframes, mobile devices, and embedded devices.

Developed by Linus Torvalds in 1991, it originated as an idea to improve the UNIX OS.

Linux OS Structure

The Linux system is built in layers, from the physical hardware to the applications you use. This separation makes the system modular and secure.

Where is Linux Used?

Linux is one of the most versatile operating systems in the world. You can find it in many different settings, from tiny embedded devices to massive supercomputers.

Desktop OS

For personal productivity computing, with distributions like Ubuntu and Linux Mint.

Headless Server OS

For remotely managed servers that don't need a graphical user interface (GUI).

Embedded Devices

For appliances and systems with limited computing functions, like smart TVs and routers.

Network OS

For routers, switches, and domain name system (DNS) servers.

Software Development

A favorite for developers, with tools like git, vim, emacs, and compilers for every language.

Cloud OS

Major cloud providers offer countless Linux instances for servers, desktops, and services.

Core Concepts

Understand the key components that make up a complete Linux system and the shell that lets you interact with it.

Components of the OS

The Shell

The **Shell** is a special program that provides an interface for you to use the operating system's services. It accepts human-readable commands (like `ls` or `cd`) and converts them into something the kernel can understand.

When you log in or open a terminal, you are interacting with a shell. While Linus Torvalds developed the Linux *kernel*, the complete Linux system also relies on GNU system utilities, libraries, and a shell.

Common Shell Types

  • BASH (Bourne Again SHell): The most widely used and default shell in most Linux systems and macOS.
  • CSH (C Shell): Uses syntax similar to the C programming language.
  • KSH (Korn SHell): Was the base for the POSIX Shell standard.

Shell Scripting Basics

A shell script is simply a file containing a list of commands that the shell can execute. This is powerful for automating repetitive tasks, performing system backups, monitoring, and much more.

How to Write a Script

  1. Create a file with a `.sh` extension (e.g., `myscript.sh`).
  2. Add the "Shebang" as the very first line. This tells the system which shell to use. 
    #!/bin/bash
  3. Add commands and comments (comments start with `#`). 
    # This is a comment
echo "Hello, World!"
  4. Make the script executable using `chmod`. 
    chmod 755 myscript.sh
  5. Run the script from your local directory. 
    ./myscript.sh

Understanding `chmod`

The `chmod 755` command sets permissions. The numbers represent **User**, **Group**, and **Public**.

Value Permission Meaning (Binary)
0 --- No Permission (0+0+0)
1 --x Execute (0+0+1)
2 -w- Write (0+2+0)
3 -wx Write + Execute (0+2+1)
4 r-- Read (4+0+0)
5 r-x Read + Execute (4+0+1)
6 rw- Read + Write (4+2+0)
7 rwx All Permissions (4+2+1)

So, chmod 755 means:

Operators are symbols that tell the shell to perform specific operations, such as arithmetic or logical comparisons. Use the tabs below to explore the different types.

Arithmetic Operators

Used for math operations. The modern way is to use $(( ... )).

A=10
B=20
echo "Sum is: $(( $A + $B ))" # Outputs 30

Control Flow

Control flow structures like loops and conditionals allow you to direct the order in which your script's commands are executed.

if / elif / else

Executes blocks of code based on conditions. Use [[ ... ]] for modern, safer tests.

read -p "Enter a number: " X

if [[ $X -gt 100 ]]; then
    echo "X is greater than 100"
elif [[ $X -eq 100 ]]; then
    echo "X is exactly 100"
else
    echo "X is less than 100"
fi

case

A cleaner way to handle multiple if statements when checking one variable against multiple patterns.

read -p "Enter 'start' or 'stop': " ACTION

case "$ACTION" in
    "start")
        echo "Starting service..."
        ;;
    "stop")
        echo "Stopping service."
        ;;
    "status"|"check")
        echo "Checking status..."
        ;;
    *) # Default (catch-all) case
        echo "Usage: start|stop|status"
        ;;
esac

for loop

Executes a block of code for each item in a list.

# 1. Loop over a list of strings
for NAME in "Amar" "Teach" "This"; do
    echo "Name: $NAME"
done

# 2. Loop over a range of numbers (Bash brace expansion)
for i in {1..5}; do
    echo "Number: $i"
done

while loop

Executes a block of code *as long as* a condition is true.

A=0
while [[ $A -lt 5 ]]; do
    echo $A
    A=$(( $A + 1 ))
done

until loop

The opposite of while. Executes a block of code *until* a condition becomes true.

A=0
until [[ $A -eq 5 ]]; do
    echo $A
    A=$(( $A + 1 ))
done

Advanced Scripting Concepts

These concepts help you write more flexible and robust scripts that can accept arguments and handle success or failure.

Positional Parameters

These are variables that hold the arguments passed to your script when you run it. For a command like ./script.sh arg1 arg2:

ParameterWhat It References
$0The name of the script itself (e.g., ./script.sh)
$1The first argument (e.g., arg1)
$2The second argument (e.g., arg2)
${10}Arguments after 9 must use braces.
$#The total number of arguments (e.g., 2)
$*All arguments as a single string (e.g., "arg1 arg2")
$@All arguments as a list of separate strings (e.g., "arg1" "arg2")

Key difference: Always use "$@" in loops to handle arguments with spaces correctly!

# This loop handles "arg 1" and "arg 2" correctly
for arg in "$@"; do
    echo "Argument: $arg"
done

Exit Status

Every command you run finishes with an **exit status** (a number between 0 and 255). This is how the shell signals success or failure.

  • 0 = Success
  • 1 (or any non-zero) = Failure

The special variable $? holds the exit status of the *last command that just ran*.

grep "some_text" file.txt
echo $? # Will be 0 if 'some_text' was found, 1 if not

if [[ $? -ne 0 ]]; then
    echo "Grep failed to find the text!"
fi
